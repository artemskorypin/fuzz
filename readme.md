# README — Краткое резюме словарей для фаззинга (SQLi)

Ниже — краткие описания каждого `.txt`-файла, для чего он нужен, какие типы уязвимостей и какие СУБД покрывает. Используй эти заметки как шпаргалку при выборе payload-набора в ZAP.

---

## sqli_general.txt

**Что:** Базовый универсальный набор SQL-payload'ов — простые кавычки, типичные `OR 1=1`, union, попытки `xp_cmdshell` и др.
**Когда использовать:** Быстрый общий проход по параметру — хороший «smoke test».
**Цель / СУБД:** Универсально (MySQL, MSSQL, Oracle — базовые шаблоны).
**Примечание:** Набор «низкошумный» → подходит для начальной проверки.

---

## mysql_basic.txt

**Что:** Простые и часто работающие payload'ы, ориентированные на MySQL (и похожие синтаксисом СУБД).
**Когда использовать:** Если есть подозрение, что бэкенд — MySQL / MariaDB.
**Цель / СУБД:** MySQL / MariaDB (инъекции с `'`/`1=1` и простые union-вставки).
**Примечание:** Хорош для быстрых проверок и обучения.

---

## mysql_blind.txt

**Что:** Payload'ы для «слепых» атак и работы с функциями вроде `load_file()`; включает конструкции для чтения файлов.
**Когда использовать:** Когда ответы сервера не содержат явных SQL-ошибок/результатов (blind SQLi).
**Цель / СУБД:** MySQL (blind + file read).
**Примечание:** Агрессивнее; может создавать много трафика/ошибок.

---

## mssql_common.txt

**Что:** Набор payload'ов, часто применимый к MS SQL Server (включая `@@version`, union для `sysobjects`, `xp_cmdshell`).
**Когда использовать:** При подозрении на MSSQL или смешанных приложениях, где возможен MS-SQL бэкенд.
**Цель / СУБД:** MSSQL (частично совместим с другими СУБД).
**Примечание:** Содержит потенциально «опасные» команды (xp_cmdshell) — запускать только с разрешения.

---

## oracle.txt

**Что:** Oracle-ориентированные payload'ы — `utl_http`, `utl_inaddr`, `v$version`, конструкции с `ROWNUM`.
**Когда использовать:** Если целевая СУБД — Oracle или есть признаки Oracle (например `v$` / `ROWNUM`).
**Цель / СУБД:** Oracle.
**Примечание:** Очень специфично; много похожих шаблонов — можно сократить по задаче.

---

## sqli_passive.txt

**Что:** Длинный набор «пассивных» и универсальных строк: различные вариации `OR`, `UNION`, функции для чтения файлов и обхода фильтров.
**Когда использовать:** Глубокий проход после общего теста; хорошо для нахождения менее тривиальных векторов.
**Цель / СУБД:** Универсально (разные СУБД).
**Примечание:** Шумный; даст много ложных срабатываний — фильтруй результаты.

---

## sqli_active.txt

**Что:** Активные payload'ы, изменяющие БД/создающие пользователей/выполняющие команды (CREATE, DROP, INSERT, xp_cmdshell и пр.).
**Когда использовать:** ТЕСТЫ НА ПЕНТЕСТОВСКОМ стенде или с явным разрешением.
**Цель / СУБД:** Зависит от строки (MySQL/MSSQL/Oracle похожие команды).
**Примечание:** Очень опасно на проде — может нарушить систему. Использовать только на стендах.

---

## mssql_blind_waitfor.txt

**Что:** Тайминговые blind-payload'ы для MSSQL (`WAITFOR DELAY` и похожие конструкции).
**Когда использовать:** Если ответ приложения не меняется, но можно измерять задержки (time-based blind SQLi).
**Цель / СУБД:** MSSQL (тайминговый вектор).
**Примечание:** Работает медленно — учитывай лимиты и таймауты.
